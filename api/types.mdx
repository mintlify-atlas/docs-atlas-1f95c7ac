## Overview

This page documents the core types, interfaces, and enums used throughout the 1inch Cross-Chain SDK.

## SDK Configuration

### CrossChainSDKConfigParams

Configuration parameters for initializing the Cross-Chain SDK.

```typescript
type CrossChainSDKConfigParams = {
  url: string
  authKey?: string
  blockchainProvider?: BlockchainProviderConnector
  httpProvider?: HttpProviderConnector
}
```

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `url` | `string` | Yes | API endpoint URL |
| `authKey` | `string` | No | Authentication key for API access |
| `blockchainProvider` | `BlockchainProviderConnector` | No | Blockchain provider connector |
| `httpProvider` | `HttpProviderConnector` | No | HTTP provider connector |

## Quote Parameters

### QuoteParams

Parameters for requesting a cross-chain quote.

```typescript
type QuoteParams<
  SrcChain extends SupportedChain = SupportedChain,
  DstChain extends SupportedChain = SupportedChain
> = {
  srcChainId: SrcChain
  dstChainId: DstChain
  srcTokenAddress: string
  dstTokenAddress: string
  amount: string
  walletAddress: string
  enableEstimate?: boolean
  permit?: string
  integratorFee?: IntegratorFeeRequest
  source?: string
  isPermit2?: boolean
}
```

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `srcChainId` | `SupportedChain` | Yes | Source chain ID |
| `dstChainId` | `SupportedChain` | Yes | Destination chain ID |
| `srcTokenAddress` | `string` | Yes | Source token address |
| `dstTokenAddress` | `string` | Yes | Destination token address |
| `amount` | `string` | Yes | Amount to swap (in token units) |
| `walletAddress` | `string` | Yes | User's wallet address |
| `enableEstimate` | `boolean` | No | Enable gas estimation |
| `permit` | `string` | No | Permit signature |
| `integratorFee` | `IntegratorFeeRequest` | No | Integrator fee configuration |
| `source` | `string` | No | Source identifier |
| `isPermit2` | `boolean` | No | Use Permit2 |

### QuoteCustomPresetParams

```typescript
type QuoteCustomPresetParams = {
```

## Order Parameters

### OrderParams

Parameters for creating a cross-chain order.

```typescript
type OrderParams = {
  walletAddress: string
  hashLock: HashLock
  secretHashes: string[]
  permit?: string
  receiver?: string
  preset?: PresetEnum
  nonce?: bigint
  source?: string
  isPermit2?: boolean
  customPreset?: CustomPreset
}
```

| Property | Type | Required | Description |
|----------|------|----------|-------------|
| `walletAddress` | `string` | Yes | User's wallet address |
| `hashLock` | `HashLock` | Yes | Hash lock for the order |
| `secretHashes` | `string[]` | Yes | Array of secret hashes |
| `permit` | `string` | No | Permit signature (without first 20 bytes) |
| `receiver` | `string` | No | Receiver address (defaults to walletAddress) |
| `preset` | `PresetEnum` | No | Preset type (defaults to recommended) |
| `nonce` | `bigint` | No | Unique nonce for the wallet |
| `source` | `string` | No | Source identifier |
| `isPermit2` | `boolean` | No | Use Permit2 |
| `customPreset` | `CustomPreset` | No | Custom preset configuration |

## Preset Types

### PresetEnum

Available preset options for cross-chain swaps.

```typescript
enum PresetEnum {
  fast = 'fast',
  medium = 'medium',
  slow = 'slow',
  custom = 'custom'
}
```

| Value | Description |
|-------|-------------|
| `fast` | Fastest execution with higher costs |
| `medium` | Balanced speed and cost |
| `slow` | Lower cost with longer execution time |
| `custom` | User-defined custom preset |

### CustomPreset

Configuration for a custom preset.

```typescript
type CustomPreset = {
  auctionDuration: number
  auctionStartAmount: string
  auctionEndAmount: string
  points?: CustomPresetPoint[]
}
```

### CustomPresetPoint

```typescript
type CustomPresetPoint = {
  toTokenAmount: string
  delay: number
}
```

## Order Status

### OrderStatus

Possible statuses for a cross-chain order.

```typescript
enum OrderStatus {
  Pending = 'pending',
  Executed = 'executed',
  Expired = 'expired',
  Cancelled = 'cancelled',
  Refunding = 'refunding',
  Refunded = 'refunded'
}
```

| Status | Description |
|--------|-------------|
| `Pending` | Order is pending execution |
| `Executed` | Order has been successfully executed |
| `Expired` | Order has expired |
| `Cancelled` | Order has been cancelled |
| `Refunding` | Order is being refunded |
| `Refunded` | Order has been refunded |

### FillStatus

Status of an order fill.

```typescript
enum FillStatus {
  Pending = 'pending',

  Refunding = 'refunding',
  Refunded = 'refunded'
```
```typescript
}
```

## Fee Configuration

### IntegratorFeeRequest

Integrator fee parameters for SDK requests.

```typescript
type IntegratorFeeRequest = {
  receiver: EvmAddress  // Address which will receive integrator's portion
  value: Bps            // Fee in basis points (1% = 100 bps)
}
```

### IntegratorFeeResponse

Integrator fee parameters from API response.

```typescript
type IntegratorFeeResponse = {
  receiver: EvmAddress  // Address receiving fee share
  value: Bps            // Total fee in basis points
  share: Bps            // Integrator's share of the fee
}
```

### TakingFeeInfo

```typescript
type TakingFeeInfo = {
  takingFeeBps: number        // 100 = 1%
  takingFeeReceiver: string
}
```

### AddressForChain

Utility type that returns the appropriate address type for a chain.

```typescript
type AddressForChain<Chain extends SupportedChain> =

    ? EvmAddress
    : Chain extends SolanaChain

      : never
```

## Example Usage

```typescript
import { QuoteParams, OrderParams, PresetEnum, OrderStatus } from '@1inch/cross-chain-sdk'

// Request a quote
const quoteParams: QuoteParams = {
  srcChainId: 1,
  dstChainId: 137,
  srcTokenAddress: '0x...',
  dstTokenAddress: '0x...',
  amount: '1000000000000000000',
  walletAddress: '0x...'
}

// Create an order with medium preset
const orderParams: OrderParams = {
  walletAddress: '0x...',
  hashLock: hashLock,
  secretHashes: ['0x...'],
  preset: PresetEnum.medium
}
```
